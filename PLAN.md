 四、L4 策略与风控层（新增）

 架构图明确要求 Policy Engine，这是之前方案缺失的关键层。

 规则引擎设计

 // policy-engine/src/types.rs

 pub struct Policy {
     pub id: Uuid,
     pub name: String,
     pub rules: Vec<Rule>,
     pub action: PolicyAction, // Approve / Deny / RequireApproval
 }

 pub enum Rule {
     MaxAmount { token: String, limit: U256 },          // 单笔限额
     DailyLimit { token: String, limit: U256 },         // 日限额
     RateLimit { max_tx: u32, window_secs: u64 },       // 频率限制
     WhitelistOnly { addresses: Vec<Address> },          // 白名单
     BlacklistCheck { addresses: Vec<Address> },         // 黑名单
     TimeWindow { start_hour: u8, end_hour: u8 },       // 时间窗口
     ChainRestriction { allowed_chains: Vec<u64> },      // 链限制
     ContractInteraction { allow_unknown: bool },        // 合约交互
 }

 pub enum PolicyAction {
     Approve,                                            // 自动放行
     Deny { reason: String },                           // 拒绝
     RequireApproval { approvers: Vec<UserId>, threshold: u32 }, // M-of-N
     RequireBiometric,                                  // 需生物识别
     Delay { seconds: u64 },                            // 延迟执行
 }

 交易审批流程

 用户发起交易
     │
     ▼
 Policy Engine 评估所有规则
     │
     ├── 低风险 (< $100, 白名单地址)
     │   └── 自动放行 → 生物识别 → MPC 签名 → 广播
     │
     ├── 中风险 ($100-$10,000, 已知地址)
     │   └── 需生物识别确认 → MPC 签名 → 广播
     │
     ├── 高风险 (> $10,000 或未知地址)
     │   └── M-of-N 审批 → 延迟 24h → 生物识别 → MPC 签名 → 广播
     │
     └── 拒绝 (黑名单/超日限额/异常模式)
         └── 拒绝 + 通知用户 + 审计日志

 实时风控

 - 异常检测：交易金额/频率/时间偏离用户历史模式时触发告警
 - 地址筛查：对比已知恶意地址数据库（后期可接入 Chainalysis / TRM Labs）
 - 合约风险：未经验证的合约交互默认需额外确认
